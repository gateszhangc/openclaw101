# OpenClaw 教程站视觉方向

站点风格参考：`https://clawhost.cloud/`

这份文档不要求照搬对方的页面，而是提炼出适合“OpenClaw 教程站”的视觉语言、布局节奏和组件风格。

## 设计目标

教程站不能做成文档站默认皮肤，也不该做成 SaaS 定价页的翻版。

目标是两件事同时成立：

1. **一眼看上去像产品站，而不是纯博客**
2. **实际读起来又能承载长教程，不让人视觉疲劳**

## 从 `clawhost.cloud` 借什么

借这几个核心气质：

- 深色主背景
- 红橙色发光氛围
- 细密网格背景
- 首屏强标题 + 强 CTA
- 产品演示区放在首屏下半部分
- 下面按区块推进：How it Works / Features / Testimonials / FAQ

不借这些：

- 直接照抄它的 copy
- 把教程站做成销售型托管站
- 复制它的品牌图标和命名

## 视觉关键词

- 黑底，不是纯黑死板，而是带暖红发光层次的近黑
- 红橙色不是点缀，而是主情绪色
- 网格是背景骨架，不是噪音贴图
- 卡片边框要轻，发光要克制
- 字体要强对比，标题有压迫感，正文保持可读

## 建议色板

下面是适合教程站的近似设计 token：

```css
:root {
  --bg: #08090b;
  --bg-soft: #111216;
  --bg-card: rgba(17, 18, 22, 0.78);
  --grid: rgba(255, 88, 63, 0.10);
  --text: #f5f7fb;
  --text-muted: #a7adb8;
  --line: rgba(255, 255, 255, 0.10);
  --primary: #ff5a36;
  --primary-hover: #ff6d4f;
  --primary-soft: rgba(255, 90, 54, 0.18);
  --glow-top: rgba(255, 80, 56, 0.28);
  --glow-mid: rgba(133, 17, 8, 0.18);
}
```

## 背景规则

首页和系列页都不应该是单色背景。

建议采用 3 层叠加：

1. 底层近黑渐变
2. 顶部红橙色大面积模糊发光
3. 全页细网格

实现建议：

```css
background:
  radial-gradient(circle at 50% 0%, rgba(255, 88, 63, 0.28), transparent 42%),
  linear-gradient(180deg, #130707 0%, #090a0d 34%, #07080a 100%);
```

网格不要太亮，透明度控制在 0.08 到 0.12 之间。

## 版式节奏

### 1. 首屏

结构参考 `clawhost.cloud`：

- 上方是轻量导航
- 中间是单个核心承诺
- 标题下方是一句解释
- 主 CTA + 次 CTA
- 首屏下半部分直接给一个“看起来像产品”的演示区域

教程站里，这块应该换成：

- 标题：学会 OpenClaw，而不是被文档淹没
- CTA：开始陪跑 / 先看第 1 篇
- 演示区：教程路线图 + 虾滑聊天 UI 模拟 + 6 篇卡片入口

### 2. 第二屏

不要直接放文章列表。

先放一个“三步理解 OpenClaw”的区块，借用 `Three Steps` 的节奏：

- 先跑通
- 再接入口
- 再养成助手

### 3. 第三屏以后

建议顺序：

1. `How It Works / 怎么学这套教程`
2. `Why This Site / 为什么不是直接看官方文档`
3. `6 篇学习路径`
4. `学完之后你会得到什么`
5. `常见问题`

## 导航栏风格

参考 `clawhost.cloud` 的轻量粘性导航：

- 左侧 Logo + 站名
- 中间锚点导航
- 右侧一个强 CTA

教程站建议导航项：

- 怎么学
- 学习路径
- 适合谁
- 常见问题
- GitHub / 官方文档
- CTA：开始陪跑

## 首页组件规则

### Hero 标题

要求：

- 1 句主承诺
- 不超过 2 行
- 有结果感，不用技术黑话开头

好方向：

- 学会 OpenClaw，不靠硬啃文档
- 从第一次聊天，到真正养出你的 AI 助手

### CTA

主按钮参考 `clawhost.cloud` 的高亮圆角按钮：

- 高饱和红橙底
- 白字
- 圆角偏大
- 可以带一个小闪电或箭头图标

次按钮可以走描边或暗底轻发光风格。

### 演示面板

不能只是一张静态截图。

建议拆成左右两栏：

- 左：虾滑对话示意
- 右：6 篇路线卡片 / 5 大模块地图

这样既有“产品感”，又直接服务教程目标。

### 功能卡片

借鉴 `clawhost.cloud` 的 Features 区块，但内容改成教程收益：

- 白话解释，不堆术语
- 一篇一件事
- 跟官方文档对齐
- 同时覆盖上手、人格、技能、自动化
- 适合普通用户
- 能直接照着做

## 教程页风格

虽然首页可以偏强视觉，但文章页必须收敛。

建议规则：

- 保留深色背景和网格
- 正文容器更窄，保证阅读体验
- 标题区仍然有红橙 glow
- 旁白框使用低透明暖色底
- 命令行代码块更亮一点，和背景拉开层次

`虾滑的话 / 虾滑提醒 / 虾滑吐槽` 这类旁白框建议用：

- 左侧高亮竖线
- 暖红半透明底
- 边框轻发光

## 组件避免事项

- 不要上来就是默认 `Inter + 黑底 + 紫色按钮`
- 不要把网格做成强干扰纹理
- 不要把整站每个区块都塞满发光边框
- 不要做成“营销站”和“文档站”之间最平庸的中间态

## 实现优先级

如果后面开始写前端，优先顺序建议是：

1. 先把首页 Hero、三步区块、教程路径区做出来
2. 再做文章页模板
3. 最后再补 testimonials、FAQ、对比区等增强模块

## 一句话总结

这套站点风格应该给人的第一感觉是：

**“这是一个认真做过产品设计的 OpenClaw 教程站。”**

而不是：

**“这是把 Markdown 扔进默认主题里生成的文档站。”**
